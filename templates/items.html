Search:

all search fields are optional
Info:
  • Name: lowercase, no special characters e.g. minicake, mysticsword, tieriiiragepants
  • Tier: number, 0, 1, 2, 3
  • Lives: number, 1 - 150
  • Max lives: number, 1 - 150
  • Gemmed: true or false
  • Nonce: number, 5 = aqua, 6 = dark, 8 = sewer, 9 = rage, other mystics have random nonces
  • Tokens: number
  • Id: number, minecraft item id
  • Enchants: enchant name + number + optional '+' or '-' e.g. regularity3, rgm2+, sweaty
explanation and limitations:

indexes players and adds all of their items to a huge list [ignores plain arrows because there are literally 5mil of them]. before it adds their items, it deletes any items in the list that the player owned. so it is constantly refreshing this list with up-to-date items. this allows it to support any items, not just unique items like mystics. however, it means that items can either appear duplicated or missing - if a player is indexed and then trades an item to another player who is then indexed, then that item will appear twice. the system makes no attempt to distinguish whether items are unique at all. in an opposite scenario, an item can be missing altogether, and will only be caught probably the next time the current owner is indexed.

it indexes players based on some custom criteria based on the last time they were on pit. effectively if a player is on pit every day, they will get indexed every day, with the same logic scaling upwards (it's not as complex as that makes it sound). uses pitpanda'd accounts.

you can also use this as an api, by changing http://www.jojo.boats/items/search?enchant1=regularity3,enchant2=rgm to http://www.jojo.boats/api/items/enchant1=regularity3,enchant2=rgm

searches are limited by the indexes that the database has. you can't do very complex searches that use many of the search fields because the database can't search through all of the items fast enough. if a search is taking more than a few seconds then the database probably doesn't support that search with its current indexes so do a simpler one. sorry the UI is so terrible LOL
